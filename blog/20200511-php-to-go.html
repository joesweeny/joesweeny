From PHP to Go

Posted by Joe Sweeny on May 20, 2020

Up until November 2018 my commercial development experience consisted of predominantly PHP with a splatter of Vue JS thrown in. I'd spent my early development career honing my skills using PHP but the time came to broaden my horizons and as advised by a colleague I took on the challenge of learning another programming language.

I felt comfortable with server side development so should I focus my attention on learning front end or should I learn another server side language? I was advised by someone who I held in high regard to learn another server side language. Doing so would not only add another string to my bow it would improve me as a software developer and improve the way I write PHP.

With this advice on board I decided to learn Go. Having read numerous Medium articles and popular development forums Go was gaining a lot of traction and popularity so naturally felt the right direction to Go (no pun intended).

I dug into blog posts, YouTube videos and various other literature but I found the barrier to entry quite steep coming from my PHP background. I often found myself looking for PHP to Go comparison resources to help reduce this barrier so I've put this post together for anyone who finds themselves in the same position I was in.

The Go community and ecosystem is extremely good at providing documentation, examples and tutorials especially to get you off the ground. The aim of this post is to dive into common programming concepts and provide a clear comparison to help PHP developers making the leap into Go development that little bit smoother.

Dynamic vs Static Typing

PHP is a dynamic programming language although since PHP 7.0 I've seen it described as a gradually typed language. Meaning on the whole PHP is a dynamic language with some static type features. Users of PHP have the flexibility to adhere to strict conventions such as function argument type hinting and declaration of return types however on the flip side users can opt not to conform to any strict typing convention and still have a fully functional application.

PHP allows developers to dynamically declare variables of a certain type and then override the same variable with a different type.

Go on the other hand is a statically typed language with strict typing enforced. Variables are assigned a type at the time of declaration and once assigned cannot be converted or cast to another type. Attempting to do so will result in a compiler error.

Go's strict typing convention provides the over riding feeling of what you see is what you get. It reduces the need for additional type checking inside functions and provides an additional level of robustness and confidence sometimes lost in weaker typed systems.

My personal approach to PHP has always been to adopt a strict approach using explicit types. I often utilise static analysis libraries such as PHPStan to enforce this approach on my projects so for Go to enforce strict typing out of the box was not a complete culture shock to me.

On the other hand if you feel more comfortable using PHP to it's more flexible capabilities the adoption of a stricter approach is something you will need to come to terms with. The inclusion of typed properties in PHP 7.4 brings PHP a step closer to the conventions Go promotes so should PHP developers choose to utilise this convention in their projects the barrier to entry from PHP to Go could somewhat be reduced.

Class vs Struct

Classes in PHP are blueprints of objects that contain properties in the form of internally scoped variables with class behaviour defined in methods.
